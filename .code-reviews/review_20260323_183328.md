+# src/data/base.py
+from abc import ABC, abstractmethod
+from pathlib import Path
+from typing import List, Tuple
+
+import pandas as pd
+
+
+class BaseLoader(ABC):
+    """
+    Contrato base para todos os loaders de dados do projeto.
+
+    Por que ABC?
+    Quando tivermos loaders diferentes (CSV, Parquet, banco de dados),
+    todos vão garantir a mesma interface: quem chama sabe que pode usar
+    .load() em qualquer loader sem precisar saber como ele funciona por dentro.
+    Isso é o Princípio da Inversão de Dependência (SOLID-D).
+
+    Uso esperado:
+        class MeuLoader(BaseLoader):
+            def load(self) -> List[Tuple[str, pd.DataFrame]]:
+                ...
+    """
+
+    def __init__(self, data_dir: Path) -> None:
+        self._validate_dir(data_dir)
+        self.data_dir = data_dir
+
+    @abstractmethod
+    def load(self) -> List[Tuple[str, pd.DataFrame]]:
+        """
+        Carrega dados e retorna lista de (identificador, DataFrame).
+
+        O identificador é tipicamente o nome da usina ou da estação.
+        """
+        ...
+
+    @staticmethod
+    def _validate_dir(path: Path) -> None:
+        """Garante que o diretório existe antes de tentar ler arquivos."""
+        if not path.exists():
+            raise FileNotFoundError(
+                f"Diretório de dados não encontrado: {path}\n"
+                "Verifique se os dados foram baixados e colocados no lugar correto."
+            )

✅ **APROVAÇÕES:**
- **Data leakage corrigido:** `normalize_data(X_train)` → `transform(X_test)` é a mudança mais importante do PR — bem documentada e testada
- **LabelBinarizer bug fix:** substituição por atribuição direta garante sempre 12 colunas de mês, independente dos dados
- **`_LazyLogger` proxy:** elegante, evita criação de arquivo de log em cada import (especialmente em testes)

- **`random_state` removido dos grids:** era anti-pattern — não é hiperparâmetro, é semente
- **`np.sqrt(MSE)` no lugar de `squared=False`:** correção de compatibilidade com sklearn >= 1.4
- **Cobertura de testes:** `test_single_month_still_produces_12_columns` e `test_transform_raises_before_fit` são testes de regressão valiosos

⚠️ **AVISOS:**

- **`FEATURES_TO_DROP` nunca é aplicado:** definido em `Config` e referenciado em zero lugares. Colunas como `VEL_VENTO`, `COORD_N` entrarão no modelo se existirem no CSV. Onde está o `df.drop(Config.FEATURES_TO_DROP, errors="ignore")`? Provavelmente no `DataProcessor`, antes do split.

- **`Config.LOGS_DIR` ignorado pelo logger:** `setup_logger` ainda recebe `log_dir: str = "logs"` hardcoded. A constante nova não tem efeito. Solução: `log_dir: str = str(Config.LOGS_DIR)`.

- **`DT_GRID` / `RF_GRID` são dicts mutáveis:** inconsistente com a filosofia de imutabilidade usada em `COLUMN_MAPPING`. `Config.DT_GRID["max_depth"].append(99)` muta silenciosamente. Wrapping com `MappingProxyType` resolve — mas como os valores são listas, é preciso `tuple` nos valores também, ou aceitar o trade-off.

- **Imputação de média antes do split (leakage leve):** `handle_missing_values` calcula `df[col].mean()` com dados de treino + teste juntos. O comentário no código reconhece isso parcialmente. Para TCC, aceitável, mas vale um `# TODO:` explícito para produção.

- **`test_load_ignores_corrupt_csv_and_continues`:** `\x00\x00\x00` pode não gerar exceção em todos os ambientes pandas. Um CSV com colunas erradas (ex: `"a,b,c"`) seria mais determinístico.
